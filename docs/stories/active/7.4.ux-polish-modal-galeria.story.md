## Story

**As a** usuária do catálogo LBG,  
**I want** receber feedback claro ao baixar uma foto e ter a interface do modal mais limpa e organizada,  
**so that** eu saiba quando o download foi concluído e não encontre elementos redundantes que poluem a UI.

---

## Contexto

Análise UX (@ux-design-expert) identificou 3 problemas de polish no modal de produto:

1. **Download silencioso:** o botão "Baixar foto" mostra "Baixando..." durante o processo, mas ao concluir some sem confirmar sucesso. Se ocorrer erro, usa `alert()` — inconsistente com o resto da UI.
2. **Botão de download duplicado:** existe nos dois locais — header do modal (linhas 159-169) e overlay da foto (linhas 237-263). O overlay é mais contextual e deve ser o único.
3. **Contador de posição sobrepõe a foto:** `{idx+1} / {total}` (linha 230) está sobre a imagem. Seria melhor posicioná-lo junto aos thumbnails.

---

## Acceptance Criteria

### AC1 — Toast de sucesso no download

- [ ] AC1.1: Ao concluir o download com sucesso, exibir toast inline (não `alert()`) por 2.5 segundos com texto "✓ Foto baixada"
- [ ] AC1.2: Toast posicionado abaixo da foto principal, acima dos thumbnails — dentro do fluxo do layout (não fixo na tela)
- [ ] AC1.3: Toast de erro também substituído por mensagem inline (não `alert()`) com texto "Não foi possível baixar. Tente novamente." — desaparece em 4s
- [ ] AC1.4: Estado `downloadMessage: { type: 'success' | 'error'; text: string } | null` gerencia o toast
- [ ] AC1.5: Toast de sucesso tem cor `#d1fae5` (verde suave), erro tem cor `#fef3c7` (âmbar) — consistente com o design system existente

### AC2 — Remover botão de download do header

- [ ] AC2.1: Remover o botão "Baixar foto" do header do modal (linhas 159-169 em `src/app/catalogo/page.tsx`)
- [ ] AC2.2: O botão de download no overlay da foto (canto inferior direito da imagem) é mantido como único ponto de download
- [ ] AC2.3: O estado `downloading` e a função `downloadFoto` permanecem inalterados — apenas o botão do header é removido

### AC3 — Reposicionar contador de posição

- [ ] AC3.1: Remover o contador `{idx+1}/{total}` sobreposto na foto (linha 230)
- [ ] AC3.2: Adicionar contador como texto simples abaixo dos thumbnails: `"3 / 8 fotos"` — fonte `0.72rem`, cor `#a8a29e`
- [ ] AC3.3: Contador só aparece quando `total > 1` (mesma condição atual)
- [ ] AC3.4: O texto de hint `"Use ← → ou clique nos thumbnails"` (linha 358) é mantido — pode ser consolidado com o contador na mesma linha: `"3 / 8 · Use ← →"`

### Quality Gates

- [ ] AC4: `npm run lint` passa sem novos erros
- [ ] AC5: `npm run typecheck` passa sem erros
- [ ] AC6: Download ainda funciona corretamente (arquivo baixado com nome correto)
- [ ] AC7: Navegação ← → e Esc continua funcionando

## Notas de Implementação

**Toast state:**
```ts
const [downloadMsg, setDownloadMsg] = useState<{ type: 'success'|'error'; text: string } | null>(null);

// No sucesso:
setDownloadMsg({ type: 'success', text: '✓ Foto baixada' });
setTimeout(() => setDownloadMsg(null), 2500);

// No erro (substituir alert):
setDownloadMsg({ type: 'error', text: 'Não foi possível baixar. Tente novamente.' });
setTimeout(() => setDownloadMsg(null), 4000);
```

**Contador consolidado (abaixo dos thumbnails):**
```tsx
{total > 1 && (
  <div style={{ fontSize:'0.72rem', color:'#a8a29e', marginTop:4 }}>
    {idx+1} / {total} · Use ← →
  </div>
)}
```
