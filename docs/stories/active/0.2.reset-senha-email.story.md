## Story

**As a** usuária que esqueceu minha senha,
**I want** receber um link de reset por email e definir uma nova senha,
**so that** eu possa recuperar meu acesso sem precisar contatar a Ana Paula.

---

## Acceptance Criteria

1. Link "Esqueci minha senha" visível na página `/login` abaixo do botão Entrar
2. Clicar no link exibe campo de email + botão "Enviar link" (inline na página ou modal)
3. `POST /api/auth/reset-request` aceita `{ email }` e retorna sempre `200 { ok: true }` (não revela se email existe)
4. Se email existe em `usuarios`: gerar token UUID, salvar em `auth_tokens` com `tipo='reset_senha'`, `expira_em = NOW() + 1h`
5. Email enviado via Resend com link: `{NEXTAUTH_URL}/reset-password?token={token}`
6. Tabela `auth_tokens` criada no Supabase (ver schema em Dev Notes)
7. Página `/reset-password` exibe form: Nova senha / Confirmar nova senha
8. Se token ausente na URL ao carregar `/reset-password` → redirecionar para `/login?error=link_invalido`
9. `POST /api/auth/reset-confirm` aceita `{ token, novaSenha, confirmarSenha }`
10. Validações: token existe + não expirado + não usado; `novaSenha === confirmarSenha`; `novaSenha.length >= 8`
11. Sucesso: atualiza `password_hash` em `usuarios` + marca `auth_tokens.usado = true`
12. Após reset com sucesso: redirecionar para `/login` com mensagem "Senha redefinida! Faça login."
13. Tokens de reset expirados ou já usados retornam `400 { error: "Link inválido ou expirado" }`
14. TypeScript compilando sem erros (`npx tsc --noEmit`)
15. ESLint sem erros (`npm run lint`)

## Dev Notes

### SQL — Criar tabela `auth_tokens`

Executar no Supabase SQL Editor (`fjzcypjldbxkcumydyzp`):

```sql
CREATE TABLE auth_tokens (
  id          BIGSERIAL PRIMARY KEY,
  email       TEXT NOT NULL,
  token       TEXT NOT NULL UNIQUE,
  tipo        TEXT NOT NULL CHECK (tipo IN ('reset_senha', 'magic_link')),
  expira_em   TIMESTAMPTZ NOT NULL,
  usado       BOOLEAN DEFAULT FALSE,
  criado_em   TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX ON auth_tokens (token);
CREATE INDEX ON auth_tokens (email, tipo, usado);
```

### Resend — Enviar email

```typescript
import { Resend } from 'resend';

const resend = new Resend(process.env.RESEND_API_KEY);

await resend.emails.send({
  from: process.env.RESEND_FROM_EMAIL ?? 'noreply@lbg.com.br',
  to: email,
  subject: 'Redefinir senha — La Bella Griffe',
  html: `
    <p>Olá,</p>
    <p>Clique no link abaixo para redefinir sua senha. O link expira em 1 hora.</p>
    <a href="${process.env.NEXTAUTH_URL}/reset-password?token=${token}">Redefinir senha</a>
    <p>Se você não solicitou isso, ignore este email.</p>
  `,
});
```

### Verificar token no Supabase

```typescript
const { data: tokenData } = await sb
  .from('auth_tokens')
  .select('id, email, expira_em, usado')
  .eq('token', token)
  .eq('tipo', 'reset_senha')
  .eq('usado', false)
  .gt('expira_em', new Date().toISOString())
  .single();

if (!tokenData) {
  return NextResponse.json({ error: 'Link inválido ou expirado' }, { status: 400 });
}
```
