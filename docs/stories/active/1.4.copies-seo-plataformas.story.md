## Story

**As a** time de e-commerce da La Bella Griffe,  
**I want** ter copies otimizados para SEO gerados automaticamente para cada produto em cada marketplace (Amazon, Mercado Livre, Shopee, Leroy Merlin, MadeiraMadeira),  
**so that** eu possa subir produtos nos marketplaces com textos profissionais prontos, respeitando os limites e critérios de ranqueamento de cada plataforma, sem reescrever manualmente para cada canal.

---

## Acceptance Criteria

### Banco de Dados
1. Tabela `produto_copies` criada no Supabase com schema definido abaixo
2. RLS ativo: SELECT para qualquer usuário autenticado; INSERT/UPDATE/DELETE apenas para admin e editor
3. Índice em `(sku, plataforma)` — UNIQUE constraint garante no máximo 1 copy por combinação
4. Migration SQL documentada em `docs/migrations/001-produto-copies.sql`

### Script de Geração
5. Script `scripts/generate-copies.ts` executável via `npx ts-node scripts/generate-copies.ts`
6. Flags: `--plataforma {nome}`, `--sku {SKU}`, `--dry-run`, `--estimate`, `--force`
7. Sem flags: processa todos os produtos com `descricao_marketing` preenchida × 5 plataformas
8. Input principal: `descricao_marketing` + metadados textuais do produto — **sem reenviar a imagem** (economia de ~6× no custo Anthropic vs. Vision)
9. Fallback: se `descricao_marketing` é nula, produto é pulado com log `[SKIP] SKU sem descricao_marketing — execute Story 1.2 primeiro`
10. Antes de gravar, valida limites de caracteres por plataforma (título, bullets, descrição)
11. Falha em produto/plataforma individual não interrompe o batch
12. Log de progresso: `[N/2175] LBG100IPANEMA × amazon → OK (título: 198/200 chars)`
13. Flag `--estimate` exibe custo total estimado (baseado em ~400 tokens input + ~300 output por copy, texto-only): `(N × 400 × 0.000003) + (N × 300 × 0.000015)` em USD

### Limits por Plataforma (OBRIGATÓRIO — script deve validar antes de INSERT)
14. Amazon: título ≤200 chars, 5 bullets ≤255 chars cada, descrição HTML ≤2000 chars
15. Mercado Livre: título ≤60 chars, descrição ≤4000 chars (sem bullets)
16. Shopee: título ≤120 chars (pode incluir 3–5 emojis), descrição ≤3000 chars
17. Leroy Merlin: título ≤100 chars, ≤10 características (bullets), descrição técnica ≤1000 chars
18. MadeiraMadeira: título ≤150 chars, descrição ≤2000 chars (foco em dimensões/material)

### API Endpoints
19. `GET /api/copies?sku={SKU}` retorna todos os copies do produto (array por plataforma)
20. `GET /api/copies?sku={SKU}&plataforma=amazon` retorna copy específico ou 404
21. `PATCH /api/copies/{id}` permite edição manual de qualquer campo exceto `sku` e `plataforma` (admin/editor)
22. `POST /api/copies/generate` recebe `{ sku: string }` no body, gera (ou re-gera) os 5 copies para aquele SKU e salva no banco; retorna array de copies gerados
23. Todos os endpoints requerem JWT válido (qualquer role para GET; admin/editor para PATCH e POST)

### Interface Admin
24. Página `/admin/copies` lista produtos com copies gerados (filtro por plataforma)
25. Para cada produto: exibir copies por plataforma com indicador de caracteres usados/máximo
26. Botão "Editar" abre modal inline para edição manual do copy
27. Botão "Gerar" por produto dispara `POST /api/copies/generate` com o SKU do produto
28. Acesso restrito a admin

### Quality
29. TypeScript compilando sem erros (`npm run typecheck`)
30. ESLint sem erros (`npm run lint`)

## Dev Notes

### Schema da Tabela `produto_copies`

A tabela usa UNIQUE `(sku, plataforma)` — no script, usar `upsert` com `onConflict`:

```typescript
await sb.from('produto_copies').upsert({
  sku, plataforma, titulo, bullets, descricao, palavras_chave,
  atualizado_em: new Date().toISOString()
}, { onConflict: 'sku,plataforma' });
```

### Padrão de Route Handler com Verificação de Role

```typescript
// Para PATCH (somente admin/editor):
const user = await verifyToken(req);
if (!user) return NextResponse.json({ error: 'Não autorizado' }, { status: 401 });
if (user.role === 'viewer') return NextResponse.json({ error: 'Acesso negado' }, { status: 403 });
```

### Por que texto-only (não Vision) na Story 1.4

A Story 1.2 (enriquecimento) já enviou a imagem ao Claude e gerou `descricao_marketing` — que captura tudo que o Vision precisa saber sobre o produto visualmente.

Na Story 1.4, o Claude recebe como input:
```
nome_produto, categoria, cor_dominante, material_aparente, tags, descricao_marketing, descricao_tecnica
```

Isso é suficiente para gerar copies SEO de qualidade. Enviar a imagem novamente seria redundante e ~10× mais caro (tokens de imagem vs. tokens de texto).

### Estimativa de Custo (texto-only)

```
2.175 copies = 435 produtos × 5 plataformas
Tokens: ~400 input texto + ~300 output por copy  (vs 1.200 input c/ imagem na Story 1.2)
Claude Sonnet 4.6: $3/1M input + $15/1M output
Custo: (2175 × 400 × 0.000003) + (2175 × 300 × 0.000015) = ~$2.61 + ~$9.79 = ~$12.40 total
Comparação se fosse Vision: ~$14.36 por imagem × 5 = ~$71.80 total — 6× mais caro
```
