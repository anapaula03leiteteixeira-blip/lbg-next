## Story

**As a** usuária da plataforma LBG,  
**I want** digitar meu e-mail uma única vez na tela de login,  
**so that** eu possa usar qualquer método de acesso (senha, magic link, ou redefinição de senha) sem repetir o mesmo dado.

---

## Contexto

A tela de login (`LoginContent.tsx`) hoje mantém **3 estados de e-mail independentes**:

- `email` → formulário principal de login
- `magicEmail` → formulário de magic link (aparece ao clicar "🪄 Entrar com MagicLink")
- `resetEmail` → formulário de "Esqueci minha senha"

O fluxo real: o usuário digita o e-mail no campo principal, não consegue entrar, clica em "MagicLink" e precisa redigitar o mesmo e-mail. Idem para redefinição de senha.

**Solução:** ao expandir o formulário de magic link ou reset, pré-preencher o campo secundário com o valor do campo `email` principal (se já preenchido). Comportamento mínimo, sem refatoração estrutural do componente.

## Dev Notes

### Localização exata das mudanças

```typescript
// LoginContent.tsx — linha ~139 (botão MagicLink)
// ANTES:
onClick={() => setShowMagic(true)}

// DEPOIS:
onClick={() => { setShowMagic(true); if (email) setMagicEmail(email); }}
```

```typescript
// LoginContent.tsx — linha ~194 (botão Esqueci senha)
// ANTES:
onClick={() => setShowReset(true)}

// DEPOIS:
onClick={() => { setShowReset(true); if (email) setResetEmail(email); }}
```

### Por que não usar `useEffect`

Um `useEffect` que observa `email` e sincroniza os campos secundários causaria re-sync a cada keystroke — se o usuário já editou o campo magic/reset, o `useEffect` sobrescreveria. A abordagem correta é copiar o valor **uma vez**, no momento do `onClick`, mantendo os estados independentes depois disso.

### Arquivo a modificar

```
src/components/auth/LoginContent.tsx   ← 2 handlers de onClick
```
