## Contexto

Durante os testes do Gabi Squad v2, a Gabi documentou que o Pix Agent (product-specialist.md) usa o campo `categoria` para mapear pilares de conteúdo. O mapeamento funciona assim:

```
pastilha   → desejo, produto
cuba       → produto, educacao
acessorio  → produto, outros
...
```

Problema: a integração doc (LBG-NEXT-INTEGRATION.md §3.3) lista `"porcelanato"`, `"revestimento"` e `"torneira"` como categorias esperadas — mas essas strings não existem no enum `Categoria` da API. Produtos dessas linhas são classificados como `"outro"`, e o Pix não consegue mapear pilares para `"outro"`.

**Investigação necessária:** antes de adicionar ao enum, verificar se esses produtos realmente existem no catálogo ou se o documento estava especulando sobre futuro.

---

## User Story

Como Pix Agent (product-specialist), quero receber `categoria` com valores que eu consigo mapear para pilares de conteúdo, para que o Strat possa alocar corretamente cada produto nos slots do calendário (desejo, produto, educação, outros) sem produzir planos genéricos.

## Notas Técnicas

### Query de investigação (AC1)

```sql
SELECT categoria, COUNT(*) as total
FROM produtos
GROUP BY categoria
ORDER BY total DESC;
```

### Query para identificar mal-classificados (AC2)

```sql
SELECT sku, nome_produto, tags
FROM produtos
WHERE categoria = 'outro'
ORDER BY nome_produto;
```
