## Story

**As a** Gabi (designer/social media da La Bella Griffe),  
**I want** consultar produtos do catálogo via API com filtros relevantes para criação de conteúdo,  
**so that** eu possa buscar automaticamente nome, imagem, descrição e categoria de qualquer produto sem precisar abrir o sistema manualmente ou usar PDFs desatualizados.

---

## Acceptance Criteria

1. `GET /api/gabi/produtos` retorna array de produtos com campos: `sku`, `nome_produto`, `categoria`, `cor_dominante`, `qualidade_foto`, `image_url`, `descricao_marketing`, `tags`
2. `GET /api/gabi/produto/[sku]` retorna produto completo (todos os campos da tabela `produtos`) ou `404` com `{ error: "Produto não encontrado" }` se SKU inexistente
3. Filtros suportados em `GET /api/gabi/produtos`: `categoria`, `qualidade_foto`, `precisa_revisao` (boolean), `limit` (default 50, max 200), `offset` (default 0)
4. Autenticação JWT obrigatória via cookie `lbg_token` — qualquer role (viewer, editor, admin) tem acesso
5. Requisição sem token ou com token inválido retorna `401 { error: "Não autorizado" }`
6. Response time < 500ms para lista de até 200 produtos
7. `descricao_marketing` nula ou vazia é retornada como `null` (não omitida)
8. Endpoint registrado no middleware como rota autenticada (não pública)
9. TypeScript compilando sem erros (`npm run typecheck`)
10. ESLint sem erros (`npm run lint`)

## Dev Notes

### Padrão de Route Handler (extraído de `src/app/api/produtos/route.ts`)

```typescript
// Padrão exato a replicar:
import { NextRequest, NextResponse } from 'next/server';
import { supabaseServer } from '@/lib/supabase';
// + adicionar verificação JWT:
import { jwtVerify } from 'jose';

const SECRET = new TextEncoder().encode(process.env.NEXTAUTH_SECRET ?? 'change-me');

async function verifyToken(req: NextRequest): Promise<{ email: string; role: string } | null> {
  const token = req.cookies.get('lbg_token')?.value;
  if (!token) return null;
  try {
    const { payload } = await jwtVerify(token, SECRET);
    return { email: payload.email as string, role: payload.role as string };
  } catch {
    return null;
  }
}
```

**Nota:** O middleware já protege as rotas mas API routes devem re-verificar o JWT internamente (defense in depth). Ver padrão em `src/app/api/auth/me/route.ts`.
