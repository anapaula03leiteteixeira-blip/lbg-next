## Story

**As a** usuária do catálogo LBG,  
**I want** que a galeria de fotos de um produto exiba sempre o ângulo frontal primeiro,  
**so that** eu veja imediatamente a melhor representação do produto ao abrir o modal, sem precisar navegar pelos ângulos.

---

## Contexto

Atualmente o `ProductModal` em `src/app/catalogo/page.tsx` (linhas 80-83) ordena as fotos apenas por `qualidade_foto`:

```ts
const QUAL_ORDER = { excelente:0, boa:1, regular:2, ruim:3 };
const sorted = [...allPhotos].sort((a, b) =>
  (QUAL_ORDER[a.qualidade_foto ?? "ruim"] ?? 9) - (QUAL_ORDER[b.qualidade_foto ?? "ruim"] ?? 9)
);
```

Isso causa o problema: uma foto "frontal" de qualidade "boa" é exibida depois de uma foto "detalhe" de qualidade "excelente". O ângulo frontal é o ângulo principal de produto e deve ter precedência sobre qualidade.

**Solução:** ordenação composta — ângulo prioritário primeiro, qualidade como critério de desempate.

## Acceptance Criteria

### Lógica de Ordenação (`src/app/catalogo/page.tsx`)

- [ ] AC1: Definir constante `ANGULO_ORDER` no topo do `ProductModal` com prioridade: `frontal:0, perspectiva:1, lateral:2, superior:3, detalhe:4, conjunto:5, embalagem:6` — ângulos não mapeados recebem valor `9`
- [ ] AC2: A função `sorted` usa ordenação composta: ângulo como critério primário, `qualidade_foto` como critério de desempate (comportamento atual preservado para mesmo ângulo)
- [ ] AC3: Fotos sem `angulo` definido (`null` ou `undefined`) são tratadas como valor `9` (ficam no final)
- [ ] AC4: `QUAL_ORDER` existente não é alterado — apenas adicionado o critério de ângulo antes dele
- [ ] AC5: O índice inicial do modal continua sendo `0` (primeira foto da lista ordenada)

### Comportamento Esperado

- [ ] AC6: Produto com foto "frontal/boa" e foto "detalhe/excelente" → frontal aparece primeiro
- [ ] AC7: Produto com duas fotos "frontal" de qualidades diferentes → a de melhor qualidade aparece primeiro entre elas
- [ ] AC8: Produto com apenas fotos sem ângulo definido → ordem de qualidade preservada (comportamento atual)

## Notas de Implementação

Mudança cirúrgica — apenas 4-6 linhas de código. Não cria novos componentes nem altera props.

```ts
// Substituir o bloco sorted atual por:
const ANGULO_ORDER: Record<string, number> = {
  frontal: 0, perspectiva: 1, lateral: 2, superior: 3,
  detalhe: 4, conjunto: 5, embalagem: 6,
};
const sorted = [...allPhotos].sort((a, b) => {
  const aA = ANGULO_ORDER[a.angulo ?? ''] ?? 9;
  const bA = ANGULO_ORDER[b.angulo ?? ''] ?? 9;
  if (aA !== bA) return aA - bA;
  return (QUAL_ORDER[a.qualidade_foto ?? 'ruim'] ?? 9)
       - (QUAL_ORDER[b.qualidade_foto ?? 'ruim'] ?? 9);
});
```
