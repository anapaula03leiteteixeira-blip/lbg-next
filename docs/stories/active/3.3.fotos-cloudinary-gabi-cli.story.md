## Contexto

Com a Story 3.1 concluída, o agente Gabi já tem acesso a `image_url` Cloudinary dos produtos via `*buscar-produto {SKU}`. Esta story completa a integração: atualiza o comportamento de `*criar-post {SKU}` para usar `image_url` do lbg-next como fonte primária de imagens, e adiciona o comando `*fotos {SKU}` que lista todas as fotos do produto com metadados (ângulo, qualidade). PDFs do Google Drive deixam de ser consultados para novos posts.

---

## User Story

Como agente Gabi, quero que ao executar `*criar-post LBG100IPANEMA` a imagem venha automaticamente do Cloudinary via lbg-next (não de PDFs do Drive), e quero poder listar todas as fotos do produto com `*fotos {SKU}` para escolher a melhor para o post.

---

## Acceptance Criteria

- [ ] AC1: Comando `*fotos {SKU}` existe no agente Gabi e está documentado na saudação
- [ ] AC2: `*fotos LBG100IPANEMA` retorna lista de todas as imagens do produto com: url, angulo, qualidade_foto (formatado para leitura humana)
- [ ] AC3: Ao executar `*criar-post {SKU}`, Gabi busca automaticamente `image_url` via `/api/gabi/produto/[sku]` — sem mencionar PDF ou Drive
- [ ] AC4: Se produto tem múltiplas fotos (`imagens[]` com >1 item), Gabi apresenta as opções e pergunta qual usar antes de gerar o post
- [ ] AC5: Se `imagens[]` tem apenas 1 foto, usa automaticamente sem perguntar
- [ ] AC6: PDFs do Google Drive NÃO são consultados para novos posts (apenas legado preservado se Drive já aberto)
- [ ] AC7: Fallback: se lbg-next offline durante `*criar-post` → mensagem: "Não foi possível buscar a imagem do catálogo. Você pode fornecer uma URL de imagem manualmente ou tentar novamente."
- [ ] AC8: Scripts `upload-drive.js` e `gerar-legenda-docx.js` NÃO são modificados
- [ ] AC9: Comportamento aditivo — demais comandos de Gabi (criar-ambiente, criar-stories etc.) não são alterados

## Notas Técnicas

**Arquivo a modificar:** `squads/la-bella/agents/gabi-social.md` (projeto aiox-core-main)

**Mudanças no comportamento de `*criar-post {SKU}`:**
1. Antes de gerar arte/legenda: chamar `*buscar-produto {SKU}` internamente (reutiliza lógica da Story 3.1)
2. Se `imagens[]` tem 1 item → usa direto
3. Se `imagens[]` tem >1 item → lista opções numeradas, aguarda escolha
4. Prossegue com geração de arte usando `image_url` selecionada

**Novo comando `*fotos {SKU}`:**
```
Chama GET /api/gabi/produto/[sku]
Extrai campo imagens[]
Formata como lista:
  1. [frontal] boa qualidade — https://res.cloudinary.com/...
  2. [lateral] regular — https://res.cloudinary.com/...
```

**Endpoint utilizado:** mesmo de Story 3.1 — `GET /api/gabi/produto/[sku]`  
Não requer endpoint novo no lbg-next.
