## Contexto

Com Story 3.1 concluída, o middleware aceita X-API-Key e os endpoints `/api/gabi/*` funcionam em produção. No entanto, o documento `squads/la-bella/LBG-NEXT-INTEGRATION.md` revela que o Pix Agent (product-specialist.md) tem dois problemas que impedem o Gabi Squad v2 de funcionar:

**Problema 1 — Endpoints errados no product-specialist.md:**
O Pix aponta para `/api/products/{sku}` e `/api/products?categoria=` mas os endpoints corretos são `/api/gabi/produto/{sku}` e `/api/gabi/produtos?categoria=`.

**Problema 2 — Schema incompatível:**
A API retorna campos com nomes diferentes dos que o ProductData schema espera:
| API retorna | Pix espera (ProductData) |
|-------------|--------------------------|
| `sku` | `product_id` |
| `nome_produto` | `nome` |
| `image_url` | `imagem_url` |
| `cor_dominante` (string) | `cores_dominantes` (array) |
| _(ausente)_ | `acabamento` |
| _(ausente)_ | `dimensoes` |
| _(ausente)_ | `metadados.linha`, `metadados.aplicacao` |

**Problema 3 — Busca texto livre ausente:**
O Pix usa `?q=` para busca por nome/linha/características, mas a API não implementa esse filtro.

**Solução escolhida:** Corrigir no agente (product-specialist.md) + adicionar `?q=` na API. Não alterar schema da API (evita breaking changes em outros consumers). O Pix já tem lógica de normalização descrita em seu YAML.

---

## User Story

Como Pix Agent (product-specialist), quero consultar `/api/gabi/produto/{SKU}` e `/api/gabi/produtos` e receber dados que posso normalizar para o formato ProductData, para que os agentes Strat, Copi e Vix do Gabi Squad v2 recebam dados estruturados corretos e o pipeline de conteúdo funcione end-to-end.

### AC1 — Adicionar `?q=` em `/api/gabi/produtos`

```typescript
// src/app/api/gabi/produtos/route.ts — adicionar após categoriaParam:
const qParam = searchParams.get("q");

// Na query Supabase, adicionar:

```

### AC2 — Suporte a `?page=`

```typescript
const pageParam = searchParams.get("page");
const page = pageParam ? Math.max(parseInt(pageParam, 10) || 1, 1) : null;
const offset = page ? (page - 1) * limit : Math.max(parseInt(offsetParam ?? "0", 10) || 0, 0);
```

### AC7 — Normalização no product-specialist.md

Adicionar bloco `normalizacao_api` no YAML do agente documentando o mapeamento:
```yaml
normalizacao_api:
  campos:
    sku → product_id
    nome_produto → nome
    image_url → imagem_url
    cor_dominante → cores_dominantes: [valor]  # string → array
    material_aparente → acabamento (quando aplicável)
    # campos sem equivalente na API → null
    dimensoes: null
    metadados.linha: null
    metadados.aplicacao: null
```
