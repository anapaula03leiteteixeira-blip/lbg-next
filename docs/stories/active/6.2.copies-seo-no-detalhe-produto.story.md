## Story

**As a** usuária do catálogo LBG,  
**I want** ver os copies SEO de um produto diretamente no seu modal de detalhe,  
**so that** eu possa verificar como o produto está anunciado em cada plataforma sem precisar navegar para `/admin/copies`.

---

## Contexto

Após o EPIC-5, o sistema gera copies SEO para 5 plataformas (Amazon, ML, Shopee, Leroy Merlin, Madeira Madeira) e os armazena em `produto_copies`. Porém, o modal de detalhe de produto (`ProductModal` em `/catalogo`) não exibe esses copies — o usuário só consegue vê-los acessando `/admin/copies`, buscando o SKU manualmente.

O objetivo desta story é adicionar uma **segunda aba "Copies SEO"** ao modal existente, sem alterar a aba "Detalhes" atual. A página `/admin/copies` permanece para operações em lote (re-gerar todos, filtrar por plataforma).

**Arquitetura aprovada:**
1. Extrair `SkuCopiesDetail` e `EditModal` de `admin/copies/page.tsx` para `src/components/copies/SkuCopiesDetail.tsx`
2. `ProductModal` ganha tabs: "Detalhes" | "Copies SEO"
3. Aba "Copies SEO" usa o componente extraído

### Tabs no ProductModal — posição no JSX

Adicionar logo após o `<div className="modal-header">`, antes do `<div className="modal-body">`:

```tsx
{/* Tabs */}
<div style={{ display: 'flex', borderBottom: '1px solid #e7e5e4', padding: '0 1.5rem' }}>
  {(['detalhes', 'copies'] as const).map(tab => (
    <button
      key={tab}
      onClick={() => setActiveTab(tab)}
      style={{
        padding: '0.625rem 1rem',
        fontSize: '0.85rem',
        fontWeight: activeTab === tab ? 600 : 400,
        color: activeTab === tab ? '#1c1917' : '#78716c',
        borderBottom: activeTab === tab ? '2px solid #b45309' : '2px solid transparent',
        background: 'none',
        border: 'none',
        cursor: 'pointer',
      }}
    >
      {tab === 'detalhes' ? '📋 Detalhes' : '🏷️ Copies SEO'}
    </button>
  ))}
</div>
```

### API endpoints utilizados

```
GET  /api/copies?sku={sku}          → lista copies do SKU (já existe)
POST /api/copies/generate           → gerar copies para um SKU (já existe)
PATCH /api/copies/{id}              → editar copy individual (já existe)
```

### Import no ProductModal

```typescript
import { SkuCopiesDetail } from '@/components/copies/SkuCopiesDetail';
```

### Fetch lazy na aba Copies

O `SkuCopiesDetail` já faz `useEffect(() => { fetch... }, [sku])` — por isso o fetch ocorre automaticamente ao montar o componente. Para garantir fetch lazy (só ao clicar na aba), o componente só é montado quando `activeTab === 'copies'`:

```tsx
{activeTab === 'copies' && <SkuCopiesDetail sku={rep.sku} />}
```

Isso garante que o modal de detalhes não faz requests extras enquanto o usuário está na aba Detalhes.
