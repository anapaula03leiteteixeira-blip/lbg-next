## Story

**As a** usuária logada no sistema (admin, editor ou viewer),
**I want** trocar minha própria senha dentro do sistema,
**so that** eu não precise depender da Ana Paula para redefinir meu acesso.

---

## Acceptance Criteria

1. `PATCH /api/auth/change-password` aceita `{ senhaAtual, novaSenha, confirmarSenha }`
2. Endpoint requer autenticação JWT — retorna `401` sem token válido
3. Se `senhaAtual` não confere com o hash atual → `400 { error: "Senha atual incorreta" }`
4. Se `novaSenha !== confirmarSenha` → `400 { error: "As senhas não coincidem" }`
5. Se `novaSenha` tem menos de 8 caracteres → `400 { error: "Senha deve ter pelo menos 8 caracteres" }`
6. Senha nova salva como bcrypt hash em `usuarios.password_hash` via UPDATE
7. Resposta de sucesso: `200 { ok: true, message: "Senha alterada com sucesso" }`
8. Modal "Alterar Senha" acessível pelo header (ícone de perfil ou menu do usuário logado)
9. Modal tem campos: Senha atual / Nova senha / Confirmar nova senha + botão "Salvar"
10. Após sucesso: modal fecha + toast de confirmação visível por 3s
11. Erros de validação exibidos inline abaixo do campo correspondente
12. TypeScript compilando sem erros (`npx tsc --noEmit`)
13. ESLint sem erros (`npm run lint`)

## Dev Notes

### Padrão de autenticação JWT (mesma função de /api/gabi/*)

```typescript
import { jwtVerify } from 'jose';
const SECRET = new TextEncoder().encode(process.env.NEXTAUTH_SECRET ?? 'change-me');

async function getAuthUser(req: NextRequest): Promise<{ email: string; role: string } | null> {
  const token = req.cookies.get('lbg_token')?.value;
  if (!token) return null;
  try {
    const { payload } = await jwtVerify(token, SECRET);
    return { email: payload.email as string, role: payload.role as string };
  } catch { return null; }
}
```

### Bcrypt (já instalado — `bcryptjs`)

```typescript
import bcrypt from 'bcryptjs';

// Verificar senha atual
const ok = await bcrypt.compare(senhaAtual, usuario.password_hash);

// Gerar novo hash
const novoHash = await bcrypt.hash(novaSenha, 10);
```

### Supabase UPDATE

```typescript
const sb = supabaseServer();
const { error } = await sb
  .from('usuarios')
  .update({ password_hash: novoHash })
  .eq('email', email);
```

### Caso especial: env-admin (id=0)

O login tem fallback para `ADMIN_EMAIL`/`ADMIN_PASSWORD_HASH` do `.env.local` — esse usuário pode não existir na tabela `usuarios`. Verificar após busca:

```typescript

  return NextResponse.json(
    { error: 'Troca de senha não disponível. Atualize a variável ADMIN_PASSWORD_HASH no servidor.' },
    { status: 400 }
  );
}
```
