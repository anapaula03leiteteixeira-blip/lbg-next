## Contexto

Com Story 5.1 concluída, o script `generate-copies.ts` tem prompts SEO otimizados para 5 plataformas. Esta story executa o script em lote para popular `produto_copies` com os 426 SKUs × 5 plataformas = 2.130 copies. O custo estimado é baixo (~$0,50-1,00 USD total). A UI `/admin/copies` já existe para validar o resultado.

**Pré-requisito operacional:** `.env.local` com `ANTHROPIC_API_KEY` e `SUPABASE_SERVICE_ROLE_KEY` válidos.

---

## User Story

Como operadora do sistema, quero executar `npx ts-node --project tsconfig.scripts.json scripts/generate-copies.ts` e ao final ter `produto_copies` populada com copies SEO para todos os produtos em todas as 5 plataformas, visíveis na UI `/admin/copies`.

---

## Acceptance Criteria

- [ ] AC1: `--estimate` exibe custo estimado antes da execução real (aprovado pela usuária antes de rodar)
- [ ] AC2: Script processa todos os produtos com `nome_produto` não-nulo (~426 SKUs)
- [ ] AC3: Ao final, `produto_copies` contém ≥ 2.000 registros (426 SKUs × 5 plataformas, tolerando falhas pontuais)
- [ ] AC4: Nenhum título excede o limite da plataforma: ML ≤60, Shopee ≤58, Amazon ≤200, Leroy ≤120, Madeira ≤120
- [ ] AC5: Nenhuma descrição Leroy Merlin excede 280 chars
- [ ] AC6: Amazon: todos os registros têm `bullets` com 5 itens e `backend_keywords` preenchido
- [ ] AC7: ML: todos os registros têm `ficha_tecnica_hints` (array de 3 itens)
- [ ] AC8: Taxa de erro < 5% (máx 107 de 2.130 copies com falha)
- [ ] AC9: Log final exibe: total gerados, erros, custo real estimado
- [ ] AC10: UI `/admin/copies` em produção (https://lbg-next.vercel.app/admin/copies) exibe os copies gerados

## Protocolo de Execução

**Sequência obrigatória — não pular etapas:**

```bash
# 1. Estimar custo (sem executar)
cd C:\Users\DELL\Downloads\lbg-next
npx ts-node --project tsconfig.scripts.json scripts/generate-copies.ts --estimate

# 2. Após aprovação de custo — testar 1 SKU em dry-run
npx ts-node --project tsconfig.scripts.json scripts/generate-copies.ts --dry-run --sku LBG100IPANEMA

# 3. Após validação visual — testar 1 SKU real (grava no banco)
npx ts-node --project tsconfig.scripts.json scripts/generate-copies.ts --sku LBG100IPANEMA

# 4. Verificar resultado no Supabase ou UI antes de rodar em lote

# 5. Executar lote completo
npx ts-node --project tsconfig.scripts.json scripts/generate-copies.ts
```

**Flags úteis para recovery:**
```bash
# Re-processar só plataforma com erros
npx ts-node --project tsconfig.scripts.json scripts/generate-copies.ts --plataforma shopee

# Re-processar SKU específico com erro
npx ts-node --project tsconfig.scripts.json scripts/generate-copies.ts --sku {SKU} --force

# Re-gerar todos (se quiser atualizar com prompts revisados)
npx ts-node --project tsconfig.scripts.json scripts/generate-copies.ts --force
```

## Validação Pós-Execução

```sql
-- Contar copies gerados
SELECT plataforma, COUNT(*) as total FROM produto_copies GROUP BY plataforma ORDER BY plataforma;

-- Verificar títulos fora do limite (não deveria ter nenhum)
SELECT sku, plataforma, LENGTH(titulo) as chars, titulo
FROM produto_copies
WHERE
  (plataforma = 'mercado_livre'   AND LENGTH(titulo) > 60)  OR
  (plataforma = 'shopee'          AND LENGTH(titulo) > 58)  OR
  (plataforma = 'amazon'          AND LENGTH(titulo) > 200) OR
  (plataforma = 'leroy_merlin'    AND LENGTH(titulo) > 120) OR
  (plataforma = 'madeira_madeira' AND LENGTH(titulo) > 120);

-- Verificar descrições Leroy fora do limite
SELECT sku, LENGTH(descricao) as chars FROM produto_copies
WHERE plataforma = 'leroy_merlin' AND LENGTH(descricao) > 280;
```
